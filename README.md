# 3806ICT Assignment 1 — Backward proof search for FOL

Implementation of Hou (2021) Algorithm 2 (naïve backward proof search over
sequent calculus LK), an improved variant, a benchmark harness, and an LNCS
report. This README describes the layout of the folder and how to reproduce
the benchmark numbers cited in the report.

## Formula syntax

The parser accepts the syntax used in the 3806ICT course notes:

| Symbol | Meaning              |
|--------|----------------------|
| `~`    | negation             |
| `&`    | conjunction          |
| `\|`   | disjunction          |
| `->`   | implication          |
| `forall x.`, `exists x.` | quantifiers (lowercase identifiers are variables; uppercase are predicate / function names) |

Example: `(forall x. P(x) -> Q(x)) -> ((forall x. P(x)) -> (forall x. Q(x)))`

## Reproducing the benchmark

Requires Python 3.11+. No third-party packages.

```sh
cd "Assignment 1"
$env:PYTHONHASHSEED="0"; python -m src.bench
```

This regenerates `results/bench_results.csv` and `results/bench_summary.txt`.
Default settings: `STEP_LIMIT=300`, `FRESH_CAP=3` (matches the values cited
in the report).

On macOS/Linux, use `PYTHONHASHSEED=0 python3 -m src.bench` instead.

To run a single formula through both provers:

```sh
python -m src.smoketest_both "(forall x. P(x) -> Q(x)) -> ((forall x. P(x)) -> (forall x. Q(x)))"
```

## Improvements over the baseline

The baseline (`src/baseline.py`) is a faithful Python port of Algorithm 2
from Hou (2021, p.67). The improved prover (`src/improved.py`) layers four
improvements on top:

- **I1 — Duplicate-formula elimination.** Skip a rule application that would
  introduce a formula already present on the same side of the sequent.
- **I2 — Branch-local loop detection.** Maintain the set of ancestor sequent
  signatures along the current branch and abort a branch on revisitation.
- **I3 — Closure look-ahead.** Score applicable invertible rules by whether
  their premises immediately produce an axiom on the current branch.
- **I4 — Herbrand-guided instantiation.** Prioritise ∀L / ∃R witness terms
  already occurring with the same predicate on the opposite side of the
  sequent, then fall back to other current terms and fresh symbols.

The baseline and improved provers remain separate so the benchmark can compare
them directly.
